## 1. 개요 (Overview)

### 1.1 배경 및 문제 정의
현대인은 식단 관리의 중요성을 알면서도, 개인 맞춤형 영양 계산과 식단 구성에 전문 지식이 없어 실천하지 못한다. 기존 앱들은 데이터베이스 의존적이거나 일반적인 정보만 제공해 개인화가 부족하다. 본 앱은 AI를 통해 사용자의 신체 정보와 목표를 즉시 분석하고, 실행 가능한 맞춤 식단과 장보기 목록을 생성한다.

### 1.2 목표 (Goals)
- 사용자가 5분 내에 완전한 하루 식단 계획을 받을 수 있도록 한다.
- 과학적 BMR/TDEE 계산으로 신뢰도 높은 영양 목표를 제시한다.
- 기피 식재료와 알레르기를 반드시 제외한 개인화 식단을 생성한다.
- AI 피드백 코칭으로 사용자가 식단을 지속할 수 있도록 지원한다.

### 1.3 비목표 (Non-Goals)
- 서버/데이터베이스 없음 — 모든 상태는 브라우저 세션 내 유지
- 의료적 진단 및 처방 기능 없음
- 사용자 계정/로그인 기능 없음 (v1.0)
- 실시간 음식 이미지 인식 없음 (v1.0)

### 2.1 타겟 사용자
체중 관리 또는 건강한 식습관을 목표로 하는 20~50대 성인.

| 페르소나 | 특징 | 핵심 니즈 |

| 다이어터 김씨 | 30대 직장인, 운동 초보 | 쉽고 빠른 저칼로리 식단 |

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 핵심 기능 — Must Have (P0)

| FR-001 | 사용자 프로파일링 | 나이/성별/키/체중/활동수준/목표 입력 폼 |

## 5. 에이전트 흐름 (Agent Flow)

```
[사용자 입력]
      ↓
User_Profiling_Agent  →  프로필 JSON 생성
      ↓
Nutrition_Agent       →  BMR/TDEE/탄단지 계산 (순수 JS)
      ↓
Menu_Planner_Agent    →  OpenRouter AI 식단 생성
  (실패 시)           →  OpenAI GPT 폴백
      ↓
Grocery_Shopper_Agent →  재료 목록 추출
      ↓
Feedback_Coaching_Agent → 피드백 반영 재조정
```

## 6. 제약 조건

### 6.1 기술 제약
- 백엔드 없음: 모든 로직은 클라이언트(브라우저) JS로 구현
- API 키는 HTML 파일에 포함 (로컬 개발용)
- CORS: OpenRouter/OpenAI API는 브라우저 직접 호출 허용

### 6.2 Edge Case 방어 규칙 (Edge_Case_Analyzer_Skill 적용)
- 나이: 1~120세 범위 강제
- 키: 50~300cm 범위 강제
- 체중: 20~500kg 범위 강제
- 목표 칼로리 하한: 여성 1,200 kcal / 남성 1,500 kcal
- 모든 단백질원 기피 시 경고 후 진행 차단
- API 30초 타임아웃 → 자동 폴백
- 빈 필수 입력 시 단계 진행 불가

## 8. 프로젝트 규칙 (CLAUDE Rules)

### 코드 컨벤션
- 단일 `index.html` 파일로 모든 기능 구현
- CSS: CSS 변수(`:root`) 기반 디자인 시스템
- JS: 에이전트별 함수 분리 (`profileAgent`, `nutritionAgent`, `menuPlannerAgent` 등)
- 에러 처리: 모든 API 호출에 try/catch + 사용자 친화적 오류 메시지

### AI 프롬프트 규칙
- 모든 AI 응답은 한국어로 요청
- 응답 형식은 JSON으로 강제 (system prompt에 명시)
- 기피 식재료는 system prompt와 user prompt 양쪽에 명시 (이중 방어)

### 스킬 활용 기록
- PRD_Formatter_Skill → 이 CLAUDE.md 생성에 적용
- Edge_Case_Analyzer_Skill → 섹션 6.2 방어 규칙에 반영
- Implementation_Review_Skill → QA 단계에서 index.html 검증에 적용
